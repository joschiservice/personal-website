## Content should feel native

The writing experience uses the same dark glass, cool light, and restrained motion as the rest of the site. The visual system steps back around the prose so that longer articles remain comfortable to read.

### A small technical example

```ts
type PublishedPost = {
  title: string;
  slug: string;
  readingTimeMinutes: number;
};

const pathFor = (post: PublishedPost) => `/blog/${post.slug}`;
```

| Principle | Practical result |
| --- | --- |
| Typed content | Invalid frontmatter fails the build |
| Static rendering | Articles load quickly and reliably |
| Local images | Assets stay versioned with their writing |

## The publishing path stays simple

1. Add an MDX file and its images.
2. Fill in the validated frontmatter.
3. Preview the draft locally.
4. Set `draft` to `false` when it is ready.

That is the whole ritual—which is exactly the point.
